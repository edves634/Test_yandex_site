# Отчёт по тестированию вёрстки: HTML + CSS + JavaScript

## 1. HTML: семантика и структура

| Проблема | Статус | Решение |
|----------|--------|---------|
| `<html lang="en">` при русском контенте | ❌ | Заменить на `lang="ru"` |
| Отсутствует `<main>` | ⚠️ | Основной контент обернуть в `<main>` |
| Ссылки-кнопки вместо `<button>` | ⚠️ | Для действий использовать `<button>` |
| Пустые `href="#"` у `.btn__learn-more` | ⚠️ | Удалить или заменить на реальные ссылки |
| Hero-заголовок: три `span` вместо одного с `<br>` | ⚠️ | Не соответствует макету (расхождение в переносах) |

---

## 2. CSS: брейкпоинты и адаптивность

| Проблема | Статус | Решение |
|----------|--------|---------|
| Жёсткая фиксация `.container` на `max-width: 37.5rem` | ❌ | Заменить на `width: 100%` + `padding` |
| Десктопный брейкпоинт стартует с `768px` (должен с `1024px`) | ⚠️ | Сменить на `@media (min-width: 1024px)` |
| Планшетный диапазон (`768px–1024px`) не проработан | ❌ | Добавить отдельный брейкпоинт или каскад стилей |
| Отсутствует адаптация под `320px` (iPhone SE) | ❌ | Добавить `@media (max-width: 374px)` |
| Картинка `.slider__card_img` перекрывает текст (`z-index`, absolute) | ❌ | Уменьшить `z-index` или изменить позиционирование |

### Проблема с макетом Figma
- В макете отсутствуют брейкпоинты `320px`, `768px`, `1024px`
- Планшетный диапазон не проработан → высокий риск ошибок на iPad

## 3. JavaScript: критические ошибки

### 3.1. Конфликт слайдеров (`slider.js` + `media.js`)

| Проблема | Влияние |
|----------|---------|
| `slider.js` создаёт горизонтальный слайдер на мобильных, `media.js` уничтожает его на десктопе | При ресайзе кнопки слайдера перестают работать |
| `media.js` сохраняет HTML как строку и восстанавливает через `innerHTML` | Потеря всех обработчиков событий |
| При каждом ресайзе происходит полная реконструкция DOM | Тормоза, фризы, нестабильное поведение |

**Решение:** удалить `media.js`, всю адаптацию вынести в CSS Grid.

---

### 3.2. Карусель участников (`carousel.js`)

| Проблема | Влияние |
|----------|---------|
| Авто-пролистывание не отключается на мобильных (нет `mouseenter`) | На телефонах карусель невозможно остановить |
| Сброс `currentIndex = 0` при `resize` | При повороте экрана пользователь теряет текущего участника |
| Сложная формула расчёта сдвига с `gap` | Микросдвиги и дрожание карточек |

**Решение:** добавить `touchstart` для остановки авто-пролистывания, пересчитывать `currentIndex` относительно максимума, упростить формулу.

---

### 3.3. Избыточные медиа-скрипты (`media.js`)

| Функция | Что делает | Проблема |
|---------|------------|----------|
| `combineHeadings()` | Объединяет два заголовка на десктопе | Перезаписывает `innerHTML` → потеря данных |
| `transformSlider()` | Заменяет слайдер на грид-сетку | Перестройка DOM, конфликт с `slider.js` |
| `rearrangeCarouselController()` | Перемещает контроллер карусели | Изменение порядка элементов |

**Общая проблема:** все эти изменения должны быть реализованы через CSS (`display`, `order`, `grid`), а не через JS.

---

## 4. Вывод

| Категория | Оценка | Ключевые риски |
|-----------|--------|----------------|
| HTML | ⚠️ Средне | Семантика, доступность |
| CSS | ❌ Проблемы | Брейкпоинты, фиксированные ширины, перекрытие текста |
| JavaScript | ❌ Критично | Конфликт скриптов, перестройка DOM, потеря обработчиков |
| Соответствие макету | ⚠️ Частично | Отсутствие планшетного брейкпоинта в Figma + расхождения в вёрстке |

### Приоритет исправлений

1. 🔴 **Удалить `media.js`** — перенести логику в CSS
2. 🔴 **Переработать брейкпоинты** — добавить `320px`, `768px`, `1024px`
3. 🟡 **Исправить карусель** — остановка авто-пролистывания на мобильных
4. 🟡 **Поправить семантику HTML** — `lang`, `<main>`, `<button>`
5. 🟢 **Убрать фиксированные ширины** — заменить на резиновую вёрстку
