## 1. Цель и объём тестирования

- Производительность (Lighthouse для десктопа и мобильных)

## 4. Отчёт тестирования производительности и адаптивности сайта (загрузка в реальных сетях)

### Общая проблема

Сайт не оптимизирован для мобильных устройств и медленных сетей. Время полной загрузки на эмуляции **Regular 3G** при разрешении **375px** достигает **17,38 секунд**. Основные причины:

- отсутствие адаптивной загрузки изображений (нет `srcset`);
- избыточный вес графики (PNG, отсутствие WebP);
- критические ошибки (бесконечные запросы шрифта на 1920px).

### 🚨 Критическая ошибка: бесконечные (цикличные) запросы шрифта Font Awesome

На скриншоте для **1920px** зафиксирована лавина запросов к файлам:  
`fontawesome-free-5.15.1.ttf`, `fontawesome-free-5.15.2.ttf`, …, вплоть до `5.15.53.ttf` и далее.

**Причина:**  
- В CSS или HTML указан динамически формируемый путь к шрифту (например, с подстановкой номера версии), либо сервер при каждом запросе возвращает редирект на новый номер, порождая бесконечную цепочку.  
- Либо скрипт генерирует новые запросы при каждой ошибке 404.

**Последствия:**  
На разрешении 1920px страница никогда не загружается полностью. Все остальные ресурсы блокируются. Сайт полностью неработоспособен на широких экранах.

### Анализ кода и выявленные проблемы

#### 1. Отсутствие адаптивной загрузки изображений
В HTML все изображения вставлены через `<img src="...">` без `srcset` и `sizes`. Тяжёлые файлы загружаются на всех устройствах:

| Файл | Размер |
|------|--------|
| `tournament-mob-2.png` | 306 КБ |
| `hero-desk.png` | 158 КБ |
| `bg-card-mob.png` | 157 КБ |
| `tournament-section__item-img1.png` | 135 КБ |
| `hero-mobile.png` | 480 КБ |

**Последствие:** на мобильном разрешении 375px загружается полный вес десктопной графики → критическое время загрузки.

#### 2. Неоптимальные форматы изображений
Все растровые изображения в формате PNG, нет версий WebP/AVIF. Фоновое изображение `.hero` переопределяется через медиа-запросы, но это не снижает вес загружаемых PNG.

#### 3. Блокирующие ресурсы и шрифты
В CSS подключены шрифты Golos Text и Merriweather через `@import`. Параметр `display=swap` присутствует, но отсутствует предзагрузка (`<link rel="preload">`). Шрифты загружаются синхронно с CSS и задерживают отрисовку текста.

#### 4. Избыточное количество JavaScript-файлов
Загружаются 4 скрипта: `index.js`, `carousel.js`, `slider.js`, `media.js` (40–80 КБ каждый). На медленном соединении каждый файл добавляет задержку RTT. Скрипты перестраивают DOM при ресайзе (объединение заголовков, пересоздание слайдера), что нагружает браузер на мобильных устройствах.

#### 5. Отсутствие `loading="lazy"`
Ни один тег `<img>` не имеет атрибута `loading="lazy"`. Все изображения загружаются одновременно, включая те, что за пределами первого экрана (аватары участников).

## 5. Адаптивность и кросс-браузерность

### 5.1. Кросс-браузерное тестирование

| Браузер / Платформа | Найденные расхождения | Статус |
|---------------------|------------------------|--------|
| **Firefox** (125) | Отображение сетки этапов, обрезка текста в мобильной версии (размер 320px) | ⚠️ Проблемы |
| **Яндекс.Браузер** | Идентичен Chrome, но наблюдаются микросдвиги в карусели участников | 🟡 Незначительно |
| **Safari (WebKit)** через Playwright | Несоответствие блока «Этапы преображения Васюков» (текст выходит за границы, не центрируется) на 375px | ❌ Критично |

### 5.2. Адаптивность на ключевых разрешениях

#### 📱 320px (iPhone SE / Firefox)
- Заголовок не соответствует макету (размер, перенос строк).
- В блоке «Чтобы поддержать Международный васюкинский турнир» текст обрезается.
- Карточка участника турнира обрезается по ширине.
- **Вывод:** вёрстка для 320px отсутствует.

#### 📱 375px (мобильный макет)
- Шапка: высота 97px вместо ~70px – лишние 30px сдвигают контент.
- Бегущие строки содержат только 1–2 фразы (должно быть 3).
- Футер: недостаточная высота, текст прижат к низу.
- Декоративные слои (шахматы, эллипсы, марка) отсутствуют – вместо них одиночные фоновые изображения.

#### 📱 768px (планшет)
- Блок «Этапы преображения Васюков»: сетка отображается некорректно, текст в карточках обрезается справа.
- Планшетный диапазон не проработан ни в макете Figma, ни в вёрстке – высокий риск ошибок.

#### 💻 1366px (десктоп – базовая ширина)
- Основные несоответствия: высота шапки (97px вместо 64px), бегущие строки вложены в hero (должны быть после), секция поддержки турнира содержит дублирующий блок `.mob`, отсутствуют фоновые слои.
- Титул Остапа Бендера – «Чемпион мира» вместо «Гроссмейстер».

#### 💻 1920px (широкий экран)
- Контент центрирован внутри `max-width: 1366px` – боковые поля не соответствуют макету Figma.
- Логотип имеет смещение ~44px относительно макета.
- Декоративные слои (жёлтый эллипс, шахматы, город) полностью отсутствуют.

> 📸 Скриншоты проблем находятся в папке `screenshots/` (файлы: `320px_issues.png`, `768px_timeline_broken.png`, `1920px_header_only.png`, `safari_webkit_375px_timeline.png`).

## 7. Качество кода (HTML/CSS/JS)

### 7.1. HTML – семантика и доступность

| Проблема | Наличие | Рекомендация |
|----------|---------|---------------|
| `lang="en"` при русском контенте | ❌ | Заменить на `lang="ru"` |
| Отсутствует `<main>` | ⚠️ | Обернуть основной контент |
| Ссылки-кнопки вместо `<button>` | ⚠️ | Для действий использовать `<button>` |
| Пустые `href="#"` у `.btn__learn-more` | ⚠️ | Удалить или заменить на реальные ссылки |
| Контрастность текста (например, `.total-count`) | ❌ | Увеличить контраст до ≥4.5:1 |
| Кнопки карусели без `aria-label` | ❌ | Добавить доступные имена |

### 7.2. CSS – брейкпоинты, адаптивность

| Проблема | Наличие | Рекомендация |
|----------|---------|---------------|
| Фиксация `.container` на `max-width: 37.5rem` | ❌ | Заменить на `width: 100%` + паддинги |
| Десктопный брейкпоинт с `768px` (должен с `1024px`) | ⚠️ | Сменить на `min-width: 1024px` |
| Планшетный диапазон (`768–1024px`) не проработан | ❌ | Добавить стили или медиа-запросы |
| Отсутствует адаптация под `320px` | ❌ | Добавить `@media (max-width: 374px)` |
| Картинка `.slider__card_img` перекрывает текст (z-index) | ❌ | Уменьшить z-index / изменить позиционирование |

### 7.3. JavaScript – критические ошибки

| Проблема | Влияние | Приоритет |
|----------|---------|------------|
| `media.js` перестраивает DOM при ресайзе (innerHTML) | Потеря обработчиков событий, конфликт слайдеров | 🔴 Критичный |
| `slider.js` и `media.js` конфликтуют: на десктопе уничтожается слайдер | Кнопки перестают работать | 🔴 Критичный |
| Авто-пролистывание карусели не отключается на мобильных | Невозможно остановить авто-скролл | 🟡 Средний |
| Сброс `currentIndex=0` при ресайзе | Пользователь теряет позицию | 🟡 Средний |

**Рекомендация:** удалить `media.js`, всю адаптацию перенести на CSS Grid/Flex, отказаться от перестроения DOM на JavaScript.

## 9. Рекомендации по исправлению (по приоритетам)

### 🔴 Критичные / High (исправить в первую очередь)

1. **Устранить бесконечные запросы шрифта на 1920px** – проверить пути к Font Awesome, убрать динамическую генерацию версий.  
2. **Удалить `media.js`** – перенести всю адаптацию на CSS Grid/Flex.  
3. **Восстановить многослойные фоны** согласно Figma (абсолютное позиционирование, `mix-blend-mode`).  
4. **Привести шапку к макету** – высота 64px на десктопе, логотип `top:26px`; на мобилке убрать фиксированную высоту.  
5. **Исправить бегущие строки** – добавить недостающие фразы (три фразы с разделителями-кружками), вынести из hero в отдельный блок.  
6. **Переделать секцию поддержки турнира** – оставить один блок с текстом и одним изображением, удалить лишний `.mob`.  
7. **Добавить адаптивность под 320px** – медиа-запрос, резиновые отступы, перенос текста.

### 🟡 Средний приоритет

8. **Внедрить адаптивные изображения** – использовать `srcset` и `sizes`, конвертировать PNG в WebP/AVIF.  
9. **Настроить кэширование статики** (`Cache-Control`), добавить `loading="lazy"` для всех изображений.  
10. **Добавить правильные брейкпоинты** – 320px, 768px, 1024px, 1366px, 1920px.  
11. **Исправить карусель участников** – уникальные фото, круглый фон, верный титул Бендера, отключаемое авто-пролистывание.  
12. **Повысить контрастность текста** (проверить все серые элементы на белом фоне).  
13. **Добавить семантические теги** (`<main>`, `aria-label`, `lang="ru"`).

### 🟢 Низкий приоритет / улучшения

14. Уменьшить количество JS-файлов, объединить их.  
15. Встроить критические CSS-стили, остальные подгружать асинхронно.  
16. Добавить предзагрузку шрифтов (`<link rel="preload">`).

**Общая оценка качества:** ❌ **Критически неудовлетворительно** – сайт не готов к релизу.

**Плюсы:**
- Базовый Lighthouse для десктопа показывает приемлемые значения (0.87).
- Вёрстка HTML/CSS в целом читаема, основные блоки присутствуют.
- Карусель и слайдер визуально работают (игнорируя JS-конфликты).

**Минусы (критические):**
- **Бесконечные запросы шрифта на 1920px** – сайт полностью не загружается на широких экранах.
- **Фундаментальное несоответствие макету Figma** – потеряны ключевые визуальные элементы (многослойные фоны, марки, правильные титулы, бегущие строки).
- **Полное отсутствие адаптивности под 320px и планшеты** – сайт непригоден для использования на iPhone SE и iPad.
- **Серьёзные ошибки в JavaScript** – `media.js` ломает работу при ресайзе, конфликтует со слайдером.
- **Производительность на мобильных устройствах неудовлетворительная** (LCP 7.8 с, полная загрузка 17+ с на 3G).

**Итог:** сайт требует **полной ревизии вёрстки** с упором на pixel perfect по макету Figma, переработки JS-логики (удаление `media.js`), исправления бесконечных запросов шрифта и добавления резиновой/адаптивной сетки. Только после этого можно проводить повторное тестирование и готовить к выпуску.
