# Отчёт тестирования производительности и адаптивности сайта

## Общая проблема

Сайт не оптимизирован для мобильных устройств и медленных сетей. Время полной загрузки на эмуляции Regular 3G достигает **17,38 секунд** на разрешении 375px. Основная причина – отсутствие адаптивной загрузки изображений и избыточный вес графики (PNG, отсутствие WebP).

## Анализ кода и выявленные проблемы

### 1. Отсутствие адаптивной загрузки изображений

В HTML все изображения вставлены через `<img src="...">` без `srcset` и `sizes`.  
Тяжёлые файлы, загружаемые на всех устройствах:

- `tournament-mob-2.png` – 306 КБ
- `hero-desk.png` – 158 КБ
- `tournament-section__item-img1.png` – 135 КБ
- `bg-card-mob.png` – 157 КБ

**Последствие:** на мобильном разрешении 375px загружается полный вес десктопной графики → критическое время загрузки.

### 2. Неоптимальные форматы изображений

Все растровые изображения в формате PNG, нет версий WebP. Фоновое изображение `.hero` переопределяется через медиа-запросы, но это не снижает вес загружаемых PNG.

### 3. Блокирующие ресурсы и шрифты

В CSS подключены `Golos Text` и `Merriweather` через `@import`. Параметр `display=swap` присутствует, но отсутствует предзагрузка (`<link rel="preload">`). Шрифты загружаются синхронно с CSS и задерживают отрисовку текста.

### 4. Избыточное количество JavaScript-файлов

Загружаются 4 скрипта (`index.js`, `carousel.js`, `slider.js`, `media.js`) весом 40–80 КБ каждый. На медленном соединении каждый файл добавляет задержку RTT. Скрипты перестраивают DOM при ресайзе (объединение заголовков, пересоздание слайдера), что нагружает браузер на мобильных устройствах.

### 5. Отсутствие `loading="lazy"`

Ни один тег `<img>` не имеет атрибута `loading="lazy"`. Все изображения загружаются одновременно, включая те, что за пределами первого экрана (аватары участников).

### 6. Консольная ошибка

Зафиксировано сообщение:

> Request for font "Gill Sans" blocked at visibility level 3 (requires 4)

Ошибка не критична, но указывает на неоптимальное объявление семейства шрифтов в CSS – попытка использовать локальный шрифт, который браузер блокирует из соображений безопасности.

## Выводы

**Главная проблема** – отсутствие адаптивной загрузки изображений и тяжёлый вес PNG-графики.  
**Вторичные недостатки**: избыточный JavaScript, отсутствие ленивой загрузки, непредзагруженные шрифты, консольная ошибка.

## Рекомендации по исправлению

1. Перевести все изображения в WebP с сохранением PNG для старых браузеров через `<picture>`.
2. Внедрить `srcset` и `sizes` для всех тегов `<img>`, предоставив мобильные и десктопные версии.
3. Добавить атрибут `loading="lazy"` для изображений вне первого экрана.
4. Объединить 4 JS-файла в один и минифицировать. Для некритических скриптов использовать `async` или `defer`.
5. Добавить предзагрузку шрифтов через `<link rel="preload">`.
6. Настроить кэширование статических ресурсов на сервере.
7. Указать в CSS резервные шрифты без запросов к системе пользователя (убрать `Gill Sans` из списка).

**После выполнения этих мероприятий ожидается снижение времени загрузки на 3G до 3–5 секунд.**
